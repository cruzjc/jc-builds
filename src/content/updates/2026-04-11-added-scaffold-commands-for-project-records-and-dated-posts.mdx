The JC-Builds site now has repo-local scaffold commands for the repetitive content work. `pnpm new:project` creates a correctly shaped project entry, while `pnpm new:update` and `pnpm new:post` create dated update records with the expected frontmatter and a default project-page link.

The published workflow guidance now treats manual local AWS deploys as the fallback path instead of the normal routine. For regular content changes, the expected flow is scaffold the entry, review the generated MDX, run `pnpm build`, and let GitHub Actions publish after merge to `main`.
