The deploy role was already in place, but the AWS account still lacked the `token.actions.githubusercontent.com` OIDC provider needed for GitHub Actions web identity federation. That gap is now closed.

With both the provider and deploy role present, pushes to `main` can use GitHub OIDC to obtain AWS credentials, sync the static site to the tracked S3 bucket, and invalidate the tracked CloudFront distribution without additional repository-level GitHub settings.
