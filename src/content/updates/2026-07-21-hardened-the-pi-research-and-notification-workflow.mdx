Hardened the Raspberry Pi research and notification workflow around the deterministic Opening-Momentum v1 boundary.

The market schedule now uses DST-aware timers rather than fixed timezone offsets, keeping session timing and day context aligned as Eastern Time changes. Alpaca/IEX data and explicit strategy rules determine actionability: new entries remain long-only, incomplete or missing opening-range data fails closed, and failed-gap or opening-range failure behavior is limited to protective exits for held longs. Existing short positions cannot generate protective sells.

Claude remains research-only. Its scheduled runner has only web search and fetch tools; local shell, file, browser, persistence, skill, and configured MCP surfaces are disabled. It may add context to the research record, but it does not decide whether a setup qualifies and is not an order-execution path. Qualifying results are routed to notification-only alerts for human review, and near-expiry entries are skipped.

Verification passed 31 strategy tests and 13 app/context tests. The dashboard returned HTTP 200, authenticated state returned HTTP 200, and unauthenticated state and alert-ingest requests returned HTTP 401. Deterministic alerts render in a read-only dashboard section without an Execute control. No test alert was sent and no order was placed, so this verification does not claim phone delivery.
