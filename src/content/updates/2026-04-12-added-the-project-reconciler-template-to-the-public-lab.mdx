This publishes the template as a tracked JC-Builds project instead of leaving it as a local repo with internal docs only.

The key idea is the reconcile-first maintenance loop. Scheduled runs do not start by pulling and editing blindly. They first check lock state, workspace safety, fast-forward conditions, and the current evidence set before any agent step can make changes.

That makes the repo useful as a baseline for future projects that want scheduled maintenance without giving up clear boundaries, reportable behavior, or git-first durability.
