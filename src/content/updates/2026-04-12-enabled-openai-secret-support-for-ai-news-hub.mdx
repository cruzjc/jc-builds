The first AWS deployment already had the secret resource in place, but the ingest worker was still falling back to local environment-only key lookup.

This update closes that gap. The deployed Lambda now resolves the OpenAI API key from `ai-news-hub/openai-api-key` in AWS Secrets Manager, supports either a raw secret string or a JSON payload, and stays on the fallback summarization path when the secret is missing or invalid.

That makes the production setup path concrete: create the key in OpenAI, store it in the existing AWS secret, and the hourly ingest worker can start using model-backed enrichment without another infrastructure change.
