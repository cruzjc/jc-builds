The production deploy path is now wired end to end from the repository itself. The workflow carries the tracked production site URL, AWS region, bucket, and CloudFront distribution values directly, and the AWS inventory now records the dedicated GitHub OIDC deploy role used for publish operations.

That change removes the previous failure mode where GitHub Actions started but could not resolve missing repository-level variables or secrets. Routine pushes to `main` can now build the site, assume the tracked deploy role, sync `dist/` to S3, and invalidate the CloudFront distribution without extra GitHub settings work.
