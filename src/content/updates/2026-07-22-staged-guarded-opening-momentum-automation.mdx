Implemented and deployed the autonomous Opening-Momentum path without giving Claude execution authority. Claude now returns a strict, short-lived JSON shortlist containing only symbol, fixed long-candidate direction, confidence, and rationale. Deterministic code rejects price, quantity, side, order type, stop, target, timing, commands, and unknown fields, then re-fetches the Alpaca clock, account, positions, assets, IEX quotes, trades, bars, and news before any entry can qualify.

The order path was tightened around documented Alpaca behavior. A buy requires an observed opening-range breakout and a fresh executable ask inside the deterministic slippage cap, then uses a whole-share DAY limit bracket. Protective exits are independent of Claude and apply only to strategy-owned longs; nested bracket legs are canceled and confirmed, then the live quantity is re-read before a DAY market sell. Stable per-session client IDs, query-only recovery for ambiguous submissions, bounded deterministic exit retries, atomic state, live `equity - last_equity`, one entry per session, and a persistent supervisor through the 10:30 ET flatten close the duplicate-order and unmonitored-exit gaps. Unresolved exposure at 10:35 ET now produces an explicit kill-switched failure instead of a successful exit.

The Trade Approvals dashboard now labels decision-only and submitted-order notifications truthfully while keeping its human proposal flow PIN-gated. The canonical Pi strategy registry was also corrected: the CLSK options, MARA options, and daily ETF rotation jobs are `legacy_dormant` with no active schedule or broker mode.

Verification passed 65 strategy tests and 18 dashboard, context, and Claude-boundary tests on the Pi. A real scheduled Claude run produced a mode-0600 same-session signal with only approved fields. A live-source read-only smoke returned `alpaca_market_closed`, and Alpaca showed zero Opening-Momentum orders today and zero open strategy orders. No test notification or order was sent.

Autonomous live trading is not enabled yet. The new timer is installed but disabled and inactive, environment authorization remains false, and the external buy kill file is present. The remaining activation prerequisite is rotation of the previously exposed Alpaca credential; the existing notification-only timer stays active until that is complete.
