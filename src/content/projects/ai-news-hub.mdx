## Objective

AI News Hub exists to make news easier for agents to consume without forcing every integration to scrape publisher pages or reverse-engineer inconsistent feeds.

The product is intentionally machine-first. It turns allowlisted feed items into stable article records with structured summaries, topic tags, source metadata, and outbound links so an agent can query recent coverage quickly and then decide when to open the original source.

## How it works

The repo was bootstrapped from the `project-reconciler-template` and now carries both the product code and the maintenance contract:

- `apps/api` exposes the public JSON feed and lookup endpoints
- `apps/mcp` exposes the same content through read-only MCP tools
- `apps/web` provides a simple public site and `llms.txt`
- `apps/workers` handles allowlisted feed ingest, dedupe, and OpenAI-backed or fallback enrichment
- `infra` defines the AWS serverless footprint with S3, CloudFront, API Gateway, Lambda, DynamoDB, SQS, EventBridge, Step Functions, and Secrets Manager

The maintenance story is equally important. A dedicated `pi13` checkout can reconcile, refresh repo maps, run validation, and attempt low-risk fixes through a guarded maintainer wrapper instead of treating automation as a blind deploy loop.

- the repo spec, rules, schedule config, and Pi bootstrap scripts are aligned with the reconciler contract
- the AWS inventory file now records the live production identifiers

The next iteration is to move the API and MCP surfaces from bundled seed data toward DynamoDB-backed live ingest output so the public site reflects the scheduled pipeline rather than the initial seed set.
